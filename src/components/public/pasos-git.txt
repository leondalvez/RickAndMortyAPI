- Pasos para subir en GitHub -
	git init
	git remote add origin "direccion del GitHub" (Sin comillas)
	git status (Para ver el estado)
	git add . (Para guardar)
	git commit -m "Ejemplo" (muestra un msj)
	git push origin master (Sube el proyecto)
	git status (Para ver si no hay errores)

-------MERGE----------
	--> le designo una carpeta en el sistema, y desde ahí abro git bash
	git init (para inicializar)
	git pull (para traerme las ramas que quiero) //acá puedo ya ingresar al archivo y modificarlo, etc
	git checkout main --> para cambiarme a la rama main, o a cualquier rama que le indique que va a ser la principal
	-- git show-branch --> para ver las ramas con los commits -- //no lo usamos siempre
	git branch nombre_rama_nueva   | (crea rama)
				       |--> puede saltarse un paso, haciendolo con el sig comando: git	checkout -b nombre_rama_nueva
	git checkout nombre_rama_nueva | (me mueve a la rama)
	git status --> para ver donde estoy
	git add .
	git commit -m "____"
	git push origin nombre_rama_nueva  
	git pull
	git checkout main (vuelvo a la rama ppal)
	git merge nombre_rama_nueva -m "commit" --> preparo el merge y comiteo
	git status
	git merge nombre_rama_nueva  --> lo repito para asegurarme que mergeo y donde
	git push origin rama_ppal(main) -->si no funciona/sale error:
		git pull origin rama_ppal
		git diff --> acá van a salir todos los archivos/commit que no coincidan, si todo esta ok, no se ejecuta
		git add . --> si hay diferencia las agrega
		git status
		git commit -m "___"
		git push origin rama_ppal
